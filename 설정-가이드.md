# 귀나무골 사과농장 — 로그인 & 소식 글쓰기 설정 가이드

이 가이드대로 하면 **주인만 로그인해서 '농장 소식'에 글을 올리고, 다른 사람은 로그인할 수 없는** 사이트가 완성됩니다.
사이트 코드(`index.html`)는 이미 준비돼 있고, 아래 콘솔 설정(약 10분)만 하면 됩니다.

준비물: 구글 계정 하나.

---

## 1단계 · Firebase 프로젝트 만들기

1. https://console.firebase.google.com 접속 → 구글 로그인
2. **프로젝트 만들기** 클릭
3. 프로젝트 이름 입력 (예: `guinamugol-apple`) — 이 이름이 나중에 `guinamugol-apple.web.app` 주소가 됩니다
4. Google 애널리틱스는 **꺼도 됩니다**(원하면 켜도 무방) → **프로젝트 만들기**

## 2단계 · 웹앱 등록하고 설정값(config) 복사하기

1. 프로젝트 첫 화면에서 **</> (웹)** 아이콘 클릭
2. 앱 닉네임 입력(예: `홈페이지`) → **앱 등록**
   - "Firebase Hosting 설정"은 지금 체크 안 해도 됩니다(나중에 배포 단계에서 함)
3. 화면에 나오는 `firebaseConfig` 값을 확인합니다. 이렇게 생겼어요:
   ```js
   const firebaseConfig = {
     apiKey: "AIzaSy........",
     authDomain: "guinamugol-apple.firebaseapp.com",
     projectId: "guinamugol-apple",
     storageBucket: "guinamugol-apple.appspot.com",
     messagingSenderId: "1234567890",
     appId: "1:1234567890:web:abcd...."
   };
   ```
4. **`index.html`을 열어서**, 아래쪽 `▼▼▼ 여기에 ... 붙여넣으세요 ▼▼▼` 부분의 6줄을 위 값으로 바꿔줍니다.
   - `"여기에_APIKEY_붙여넣기"` → 실제 `apiKey` 값… 이런 식으로 6개 모두 교체
   - 큰따옴표(`"`)는 그대로 두고 안쪽 내용만 바꾸면 됩니다

> 참고: 이 값들은 코드에 노출돼도 괜찮은 값이에요. 실제 보안은 아래 5단계의 **보안 규칙**이 담당합니다.

## 3단계 · 로그인 기능 켜고, 주인 계정 하나 만들기

1. 왼쪽 메뉴 **빌드 → Authentication** → **시작하기**
2. **Sign-in method** 탭 → **이메일/비밀번호** 선택 → **사용 설정**(켜기) → 저장
3. **Users** 탭 → **사용자 추가**
   - 주인이 쓸 이메일과 비밀번호를 입력해 계정을 **딱 하나만** 만듭니다
   - 이게 대표님이 사이트에서 로그인할 때 쓸 아이디/비번이에요

> 핵심: 사이트에는 '회원가입' 기능이 없습니다. 그래서 여기서 만든 이 계정 말고는 **누구도 로그인할 수 없습니다.**

## 4단계 · 데이터베이스(글 저장소) 만들기

1. 왼쪽 메뉴 **빌드 → Firestore Database** → **데이터베이스 만들기**
2. 위치는 기본값(또는 `asia-northeast3 (서울)`) 선택
3. 시작 모드는 **프로덕션 모드**로 시작 → 만들기
   (규칙은 다음 단계에서 바로 넣습니다)

## 5단계 · 보안 규칙 넣기 (주인만 글쓰기 잠금)

1. 먼저 **주인 계정의 UID**를 확인합니다: **Authentication → Users** 탭에서 아까 만든 계정 줄의 **User UID**를 복사 (예: `Ab12Cd34...`)
2. **Firestore Database → 규칙(Rules)** 탭으로 이동
3. 내용을 아래로 **전체 교체**하고, `여기에_주인_UID_붙여넣기` 자리에 방금 복사한 UID를 넣습니다:

   ```
   rules_version = '2';
   service cloud.firestore {
     match /databases/{database}/documents {
       match /posts/{postId} {
         allow read: if true;
         allow write: if request.auth != null
                      && request.auth.uid == "여기에_주인_UID_붙여넣기";
       }
     }
   }
   ```
4. **게시(Publish)** 클릭

이렇게 하면 글 읽기는 누구나 가능하지만, 글쓰기·삭제는 오직 이 UID(주인)만 가능합니다. 서버(구글)에서 강제되기 때문에 코드를 조작해도 뚫리지 않아요.

---

## 6단계 · 배포해서 `.web.app` 주소 얻기

컴퓨터에서 아래를 한 번만 설치·설정하면 됩니다. (명령어는 그대로 복사해 실행)

```bash
# 1) Firebase 도구 설치 (Node.js 필요)
npm install -g firebase-tools

# 2) 로그인 (브라우저가 열리면 구글 로그인)
firebase login

# 3) 이 index.html이 있는 폴더에서 실행
firebase init hosting
#   - "Use an existing project" → 아까 만든 프로젝트 선택
#   - "public directory?" 질문에는  .  (마침표, 현재 폴더) 입력
#   - "single-page app?"  →  No
#   - 기존 index.html 덮어쓰기?  →  No (반드시 No!)

# 4) 배포
firebase deploy
```

끝나면 터미널에 `https://프로젝트이름.web.app` 주소가 나옵니다. 그게 완성된 사이트예요.
앞으로 내용을 바꾸면 `firebase deploy`만 다시 실행하면 갱신됩니다.

> 지금은 GitHub에도 올라가 있는데, Firebase Hosting으로 배포하면 `.web.app` 주소로도 접속됩니다. 둘 중 어느 쪽을 주소로 쓸지는 자유예요. `.web.app`을 쓰기로 했으니 그 주소를 명함·네이버에 넣으면 됩니다.

## 7단계 · (선택) 네이버 검색 등록

1. https://searchadvisor.naver.com 접속 → 네이버 로그인 → **웹마스터 도구**
2. 사이트 등록에 `https://프로젝트이름.web.app` 입력
3. 소유확인 방법 중 **HTML 태그**를 고르면 `<meta name="naver-site-verification" ...>` 한 줄을 줍니다
   - 이 줄을 `index.html`의 `<head>` 안(맨 위 `<title>` 근처)에 붙여넣고 다시 `firebase deploy` → 네이버에서 **확인** 클릭
4. 이후 사이트맵/URL을 제출하면 네이버 검색에 노출 대상이 됩니다
   - 농장이라면 **네이버 스마트플레이스(지도 등록)**도 함께 하면 검색 노출에 훨씬 효과적이에요

## 요금 관련 (안심하세요)

- 로그인(Authentication)과 글 저장(Firestore), `.web.app` 호스팅은 **무료 요금제로 충분**합니다.
- 무료 한도: 글 저장 1GB, 하루 읽기 5만 회 / 쓰기 2만 회 — 농장 소식용으로는 넘칠 일이 거의 없어요.
- **사진 첨부**는 이번 버전에 넣지 않았습니다. 나중에 필요하면 알려주세요(사진 저장은 카드 등록이 필요한 Blaze 요금제가 됩니다 — 한도 안 넘으면 요금은 0원).

## 사용 방법 (완성 후)

- 방문자: 그냥 사이트를 보면 됩니다. '농장 소식'에 올라온 글이 보여요.
- 주인: 맨 아래 **관리자** 글씨를 눌러 로그인 → '농장 소식'에 글쓰기 창이 나타남 → 제목·내용 쓰고 **올리기**.
  글 옆의 **삭제** 버튼으로 지울 수 있고, 다 쓰면 **로그아웃** 하면 됩니다.
